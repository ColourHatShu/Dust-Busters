# 🔬 Full-App Audit — 2026-06-28 (9-agent QA sweep)

Source: multi-agent audit (one agent per area, evidence-based). Synthesis agent
died on a connection error, so this is the knight's hand-prioritization of the 9
area reports. **Excludes** already-fixed items (action throw-crashes, onboarding
trigger, admin verify trigger, broadcast TTL, shared MessagePanel hydration,
rebroadcast idempotency).

## 🔴 High — money / security / correctness (do first)

1. **Timezone bug (pervasive).** `datetime-local` is parsed in the *server's*
   timezone. In a UTC production deploy, appointment times are stored 7–8h off and
   same-day afternoon bookings are wrongly rejected; it also skews the refund
   window. The `validateBooking` unit test masks it by feeding a UTC-suffixed
   string the form never sends. → Parse the local wall-clock pick deterministically
   (treat as the customer's tz / append no Z, or carry an explicit offset). Fix the
   test to use a form-shaped value. _(booking-create, cross-cutting)_
2. **Double-charge possible.** Payment idempotency is only on `stripe_session_id`,
   not `(booking_id, type)`, so a deposit or balance can be charged twice (e.g. the
   post-Stripe-return window re-shows the Pay button). → Unique/guard on
   `(booking_id, type)` among paid payments + hide the pay button once a paid row
   exists. _(payments, booking-lifecycle)_
3. **Refund accounting → negative "Net paid".** Refunds are double-counted, so a
   refunded cancellation shows a negative total on the receipt and the admin
   dashboard. → Treat refund rows as negative/own bucket; recompute net. _(payments,
   booking-lifecycle, admin)_
4. **`disputed` missing from 3 duplicated STATUS maps.** Renders as a raw gray
   enum on the booking detail + list, and disputed bookings vanish from the list's
   Active/Completed tabs. → Use the complete maps in `src/lib/types.ts` everywhere.
   _(booking-lifecycle)_ — note: the futuristic-UI redesign is rewriting these maps;
   reconcile after it lands.
5. **Dispute refund refunds the WRONG Stripe intent** on multi-payment bookings
   (refunds the first/most-recent instead of the disputed one). → Select the
   correct payment intent. _(admin — money)_
6. **Resolving a dispute leaves the booking stuck in `disputed`.** → Transition to
   a sane post-resolution status. _(admin)_
7. **RLS INSERT holes (REST-bypassable).** `booking_messages` and `disputes`
   INSERT policies only check the actor's own id, not booking participation — a
   user can post to others' bookings via the auto REST API. → Add participation
   checks (migration). _(cross-cutting — security)_
8. **`cleaner_details` UPDATE has no WITH CHECK.** A suspended cleaner can
   self-reactivate (`active`/`accepting_jobs`) via a direct API write, bypassing
   admin moderation (the trigger only pins `id_verified`). → Add WITH CHECK +
   pin moderation fields (migration). _(account — security)_
9. **Local duplicate `MessagePanel`.** The hydration fix landed on
   `src/components/MessagePanel.tsx` (used by the cleaner page), but the *customer*
   booking page imports a separate `src/app/bookings/[id]/MessagePanel.tsx` that
   still has the `Date.now()`/locale hydration bug. → Apply the same mount-gate.
   _(cross-cutting)_

## 🟠 Medium — UX / reachability / robustness

10. **Map freezes after "Search again":** the poll loop never resumes, so the map
    goes stale until reload. _(matching)_
11. **Cleaner profile page has no nav entry point** at all. _(cleaner-jobs)_
12. **Completed/paid jobs fall off the cleaner UI**, so the "rate the customer"
    feature is barely reachable. _(cleaner-jobs)_
13. **Admin cleaner roster "Avg Rating" column is always blank** (reads a
    nonexistent column). _(admin)_
14. **`/cleaner/onboard` is unguarded:** an existing/deactivated cleaner can
    re-run it (self-reactivate) and an admin can accidentally demote themselves to
    cleaner. _(auth)_
15. **Fully-declined broadcast lingers** in `broadcasting` for the full 24h TTL
    showing "0 deciding" instead of flipping to no_cleaner_found. _(matching)_
16. **No `aria-live` regions** for any dynamic feedback (matching status, toasts).
    _(cross-cutting — a11y)_
17. **Forgot-password completion path is missing** (request exists, reset/landing
    does not). _(auth)_
